# 审稿：提案 (pitch.md)

## 1. 结构合理性

**当前结构：**
- 为什么需要提案
- 如何写好提案
- 提案的五个要素
- 如何带着提案开牌桌会
- 反例：缺少要素的代价
- AI 时代的提案变化
- 附录：完整提案案例

**评估：** 结构完整，从"为什么"到"怎么做"到"反例"到"新趋势"，逻辑清晰。

**建议调整：**
- 附录的提案案例过长（约3000字），可以考虑放到单独的在线链接，正文只保留摘要

## 2. 深度思考：这是最好的结构吗？

接近最好。五个要素（问题、胃口、解法、陷阱、不做什么）的框架清晰实用。

可考虑的优化：

- "AI 时代的提案变化"放在正文末尾略显突兀。可以移到"如何写好提案"之后，作为方法的补充。
- 附录案例太长，建议大幅精简或移到外部链接。

## 3. 清晰度

整体清晰。五个要素的命名直观易记。

一处可加强：

- "胃口"（Appetite）这个术语第一次出现时，可以加一句解释来源（Shape Up 方法论）

## 4. 故事与案例

**优点：**
- 快捷键功能的对话很真实，展示了"胃口缺失"的问题
- 广场功能被否决的案例展示了"让反对声音提前出现"的价值
- 稍后阅读的提案模板清晰实用

**可改进：**
- Evernote 膨胀的案例可以更具体：加了哪些功能？用户怎么抱怨？

## 5. 简洁性

主要问题在附录：

附录的 AI 辅助纠错提案约 3000 字，包含大量技术细节（Prompt、成本计算、模型选择）。对大多数读者来说过于详细。

**建议：**
- 保留提案的核心结构（五要素）+ 牌桌讨论精华（约500字）
- 删除技术实现细节（Prompt、成本表、流程图）

- 圣埃克苏佩里的引用：建议加注是《小王子》作者
- 其他无明显问题
