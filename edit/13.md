# 编辑意见

[x] 强烈建议书稿交付的版本，按照严格的章节层次来呈现，并结合目录的呈现意图。如：
第一部分
第一章
1.1
1.1.1
（之后的部分可能就不上目录了，所以不用编排章节序号，但可以有一级标题，比如“1.”。
「后续排版时我们不想有这些编号，可以再统一去掉，现在只看字号大小来区分层次，给编辑和排版制造了不必要的麻烦，也容易出错。」

[] 章节一二级标题准确处理
[] 图书里不允许有外链，所有链接最终会单独放在一个文件里供读者下载，也可以考虑将链接全部去掉，改用图文描述清楚
[] 系统思维里微信的图片重绘
[] 最完整的图片使用是同时具备三要素：图的引用（放在图上方紧邻段落）、图号和图题；部分省略只能省略图题；当然也可以三要素都没有。但以目前我们图的情况，没有引用，图文关联不上，显得很突兀。如果仍然采用这些图，则可能需要引入图片使用三要素，能让读者更好理解一些
[] 在图书里，除了明确的标题，建议尽量用完整句子，不要用一个词或两个词用冒号连接这种碎片化的写作模式，并且句末应该有句号。
[] 标题尽量凝练一些，不要嵌套复杂的信息，读者能懂。而且部分、章、节应该是信息越来越完整，反过来会显得头重脚轻。
[] “1.”这种形式和“·”这种形式的功能区分有规律吗？现在改进是在混用，显得有些凌乱和不考究。(出现在精益画布)
  - 改成引用形式是否合适？
[] 这里还有一个重大的问题：过渡语。从规范和体验角度，建议任两级标题之间，任意两种体例之间（如标题和图、图和表、图和并列项、正文和并列项，等等）都最好能有过渡语，否则视觉上不好看，读者也不理解后续的划分是按照什么逻辑展开的。
[] 虽然不是一个硬性规定，但篇幅和层级最好能有一个线性关系，也就是层级高篇幅相对长一些。如果一节只有几百个字，下一级的小节反而有几千字，就稍显不协调。基于这种考虑，有时候需要分拆、合并标题以调整层级，让篇幅相对合理一些。
「非硬性要求，如有特殊情况或只是个别章节篇幅不合理，是可以接受的。」
[] 这种过度口语化的句子，也需要全书过一遍。一方面有可能不符合出版要求，另一方面也更重要，就是会让书稿显得“粗糙”。(我拿金字塔过了一遍)
[] 这种写法就明显太简练了。下面为什么有 12 个东西，这 12 个东西和上下文是什么关系。写作千万别把读者当自己，自己什么都明白，而读者是有可能什么都不明白的。需要把完整的逻辑用完整的表述说得周到一些，不能跳跃也不能抄近路。(QQ 的例子)
