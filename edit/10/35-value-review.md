# 《价值观》审阅意见

> 全书第 35 章 / 共 40 章

## Q1: 共识与非共识

**这一章说的多数是共识，但缺少真正的非共识洞察。**

共识的部分：
- 价值观要有代价才是真的（Lencioni 的那句话）
- 客户第一、负责、简单这些价值观本身
- DeepSeek、乔布斯、马斯克的案例都是公开的名人故事

**缺失的非共识视角：**

1. **小公司的价值观悖论**：本章开头问"小公司需要价值观吗"，但没有正面回答这个问题。非共识观点应该是：小公司在早期不需要成文的价值观，价值观是行为的自然沉淀，而非设计的产物。你们 2005 年创业，什么时候才正式写下这四条价值观的？这个时间点很重要，应该说清楚。

2. **价值观的成本账**：章节提到"愿意付出代价"，但没有算账。应该补充：
   - 砍掉发现页损失了多少收入？具体数字。
   - 17 分钟退款案例，公司承担了多少成本？一年有多少这样的案例？
   - 拒绝提现页广告损失了多少钱？
   - 这些代价加起来占公司收入的百分比是多少？

   没有具体数字，"付出代价"就是空话。

3. **价值观的黑暗面**：每个价值观都有代价和副作用，但本章完全没提。非共识观点：
   - Hacker Spirit 可能导致过度折腾、不务正业
   - Accountability 可能导致边界模糊、过度负责导致团队无法成长
   - Customer First 可能导致公司亏损、被客户绑架
   - Keep Simple 可能导致产品简陋、功能不足

   应该至少提一个你们踩过的坑：因为坚守某个价值观而犯的错误。

4. **HACK 的巧合还是设计**：四个价值观的首字母凑成 HACK，这是巧合还是刻意设计？如果是刻意的，说明你们在"包装"价值观以便于传播，这本身就违背了"简单"。如果是巧合，这个巧合怎么来的？这个疑问应该说清楚。

5. **价值观的优先级冲突**：四条价值观之间如果冲突，怎么选？比如：
   - 用黑客精神折腾新产品（H）vs 保持产品简单（K）
   - 客户要求复杂功能（C）vs 保持简单（K）
   - 承诺给客户的功能（A）vs 保持产品简单（K）

   应该补充一个真实案例：当价值观互相冲突时，你们怎么选择。

**建议添加的非共识观点：**

一段话放在"知识星球的价值观"之前：

"坦白说，我们创业的前几年根本没有成文的价值观。我们只是凭直觉做事。价值观是后来回顾时，从一系列决策中提炼出来的。不是先有价值观再有行为，而是先有行为再总结价值观。

如果你的公司只有三个人，不要急着写价值观。等你做了一百个决策，回头看哪些决策是你不后悔的，那些决策背后的原则，才是你的价值观。

价值观不是设计出来的，是长出来的。"

## Q2: 知识星球故事建议

本章已经有三个知识星球的故事（17 分钟退款、提现页广告、发现页），但都是"公司决策"的故事，缺少"人"的故事和"社区"的故事。

**建议补充的故事类型：**

1. **星主坚守价值观的故事**（Hacker Spirit + Accountability）
   - 某个星主拒绝了一个赚钱但违背初心的合作
   - 某个星主主动给加入时间长但不活跃的用户退款
   - 某个星主在星球里公开道歉并改正错误

   可以匿名，但要有具体细节。这类故事能说明：价值观不是公司独有的，而是会传染给用户的。

2. **用户因为知识星球的简单而留下的故事**（Keep Simple）
   - 对比其他复杂的社区产品（Discord、小红圈等），为什么选择知识星球
   - 某个知识星球的运营者说：知识星球的简单让我可以专注在内容上，而不是运营技巧上

   需要找一个具体的用户，有对比，有细节。

3. **知识星球的客服如何践行"客户第一"的故事**（Customer First）
   - 不要只讲你作为 CEO 的决策，讲一个客服或产品经理的故事
   - 某次用户投诉，客服怎么处理的？超出职责范围但还是帮用户解决了什么问题？
   - 这个故事要有对话，有细节，让人感受到"多走一步"

4. **知识星球内部的反馈文化**（Keep Simple 里的"坦诚沟通"）
   - 某个同事批评了你的产品决策，你怎么回应的？
   - 或者你批评了某个同事的方案，对方怎么反应的？
   - 这个故事要有冲突、有转折，不能是和谐美满的样板戏

5. **价值观冲突的故事**
   - 某次决策，黑客精神和简单冲突了，你们怎么选的？
   - 某次决策，客户第一和公司利益冲突了，你们怎么选的？

   这个故事很重要，因为它会暴露价值观的优先级和边界。

**最需要补充的是第 4 个和第 5 个。** 前三个是"正面案例"，后两个是"真实案例"。真实案例更有说服力。

## Q3: 图片与表格建议

本章目前只有一张图（发现页截图），视觉化程度不够。以下是具体建议：

### 建议 1：价值观的代价账本（表格）

在"知识星球的价值观"章节的最后，添加一个表格，量化价值观的代价：

| 价值观 | 具体决策 | 损失/代价 | 时间 |
|-------|---------|----------|------|
| Customer First | 17 分钟退款案例 | ¥XXX/年（类似案例 XX 起） | 2023 |
| Customer First | 拒绝提现页广告 | ¥XXX/年 | 2022 |
| Keep Simple | 砍掉发现页 | ¥XXX/年的增长机会 | 2021 |
| Customer First | 生财有术免单 | ¥XX 万 | 2020 |
| **总计** | | **¥XX 万/年（占年收入 X%）** | |

这个表格的作用：
- 量化"付出代价"，让价值观可见
- 让读者看到：你们不是说说而已
- 给其他创业者一个参考：坚守价值观的成本占比

### 建议 2：HACK 价值观的四象限图（图片）

用一个四象限图来展示四个价值观之间的关系：

```
         向外（客户/市场）
              |
              | C
              | Customer First
              |
     K -------+------- H
Keep Simple   |   Hacker Spirit
              |
              | A
              | Accountability
              |
         向内（团队/执行）
```

横轴：简单 ←→ 创新
纵轴：向内 ←→ 向外

这个图的作用：
- 让四个价值观的关系可视化
- 说明：C 和 H 是外向的（面向客户和市场），K 和 A 是内向的（面向产品和团队）
- 说明：H 和 K 是矛盾的（创新 vs 简单），需要平衡

### 建议 3：价值观的行为清单（表格）

在每个价值观的小节后面，添加一个"行为清单"，让价值观可操作：

**H - Hacker Spirit**

| ✓ 符合黑客精神的行为 | ✗ 违背黑客精神的行为 |
|-------------------|-------------------|
| 主动尝试新技术 | 拒绝学习新工具 |
| 开源自己的代码 | 只关心自己的 KPI |
| 找到不一样的解决方案 | 照搬竞品 |
| 质疑现有方案 | 因为"一直这么做"就不改 |

其他三个价值观也类似。这个清单的作用：
- 让价值观具体化
- 让团队成员可以对照自查
- 让价值观从"墙上的标语"变成"日常的行为指南"

### 建议 4：用 DeepSeek 的故事配图

DeepSeek 的故事很长（50+ 行），可以用一张信息图来总结：

```
OpenAI GPT-4          vs          DeepSeek R1
训练成本: $100M+                  训练成本: $5.6M
芯片: 最先进                      芯片: 受禁运限制
模式: 闭源                        模式: 开源
结果: 顶级性能                    结果: 相当或超越

关键: 用更聪明的算法弥补硬件不足
```

这个图的作用：
- 让对比更直观
- 让"黑客精神"的威力可见
- 提升阅读体验（长段落需要视觉间隔）

### 建议 5：砍掉发现页的前后对比图

现在只有一张发现页的截图。应该补充：
- 砍掉前：底部四个 tab（星球、发现、动态、我）
- 砍掉后：底部三个 tab（星球、动态、我）

前后对比，让"简单"的效果可见。

## Q4: 整体评分

**评分：72/100**

### 扣掉的 28 分，扣在哪里：

**-8 分：缺少非共识洞察**
- 全章都是"正确的话"，但缺少反常识的观点
- 没有回答"小公司需要价值观吗"这个开头的问题
- 没有暴露价值观的黑暗面和代价

- 段永平的故事、Netflix 的故事、马斯克的故事，都是名人案例，缺少"你自己"的故事
- 知识星球自己的故事只有三个（17 分钟退款、提现页广告、发现页），太少

**-5 分：缺少关键数据**
- "付出代价"没有量化，不知道代价有多大
- "二十多年准时发工资"，没说公司多少人、营收多少、利润多少，无法判断这个"准时"的难度

**-4 分：价值观的冲突和优先级不清晰**
- 四个价值观之间如果冲突，怎么选？没说
- HACK 这个缩写是巧合还是设计？没说
- 价值观什么时候写下来的？怎么迭代的？没说

**-3 分：结构松散，三个小节之间的逻辑不清晰**
- 第一节"二十年我们做对了什么"，是回忆
- 第二节"知识星球的价值观"，是陈述
- 第三节"迭代"，是建议
- 三节之间缺少承接，读起来像三篇文章拼在一起

**-1 分：开头的问题没有回答**
- 开头问"小公司需要价值观吗"，但全章都在讲"价值观是什么"和"怎么践行"，没有回答"需不需要"
- 如果不需要，为什么要写这一章？如果需要，什么时候需要？多小的公司算小？
- 这个问题很重要，但被忽略了

### 值得肯定的地方：

**+5 分：案例选择好**
- DeepSeek 的案例很新（2025 年 1 月），切中时事
- 乔布斯、Netflix、马斯克的案例都是经典案例，说服力强
- 绿盟科技的案例虽然久远，但"自组织"的场景感很强

**+5 分：写作风格符合要求**
- 短句，简洁，说人话
- 段落紧凑，每段不超过 8 句
- 没有空话套话，每句话都在说事

**+3 分：引言选得好**
- Patrick Lencioni 和 Roy Disney 的两句话，一针见血
- 放在开头，定下了全章的基调

**总体评价：**

这是一章"安全"的文章。没有大错，但也没有大亮点。读完之后，读者会觉得"嗯，价值观很重要"，但不会觉得"原来价值观可以这样用"。

缺少的是：
1. 非共识的洞察
2. 真实的冲突和挣扎
3. 量化的代价
4. 可操作的指南

如果这章能补上这四点，可以到 85 分以上。

## Q5: 必要性审视

**砍掉这一章，对全书是减色的。但如果不大改，这一章的价值不大。**

### 为什么砍掉会减色：

1. **全书结构需要这一章**
   - 目录显示，第四部分"产品之外"有两个子部分：经营问题和组织问题
   - 组织问题包括：人才密度、价值观、工作与生活平衡、远程办公、AI
   - 价值观是组织问题的核心，砍掉会导致组织部分不完整

2. **前面章节埋了伏笔**
   - 第一部分讲"黑客思维"，第 35 章讲"黑客精神"，前后呼应
   - 前面多次提到知识星球的决策（如"简单""客户第一"），这一章是集中解释
   - 砍掉会导致前面的伏笔成为悬念

3. **读者期待**
   - 读到第 35 章，读者已经看了大量产品和增长的方法论
   - 这时候读者会想：这家公司为什么能坚持这些方法论？
   - 答案是：因为价值观
   - 砍掉会让读者觉得"只讲方法，不讲原则"

### 为什么说"如果不大改，价值观不大"：

1. **现在的内容太"正确"**
   - 读完之后，读者会觉得"你说的都对"，但不会觉得"我学到了新东西"
   - 价值观这个话题，已经被讲烂了
   - 如果只是把"客户第一""简单""负责"这些共识再说一遍，不如不写

2. **缺少真实的挣扎**
   - 价值观最有价值的部分，不是"是什么"，而是"怎么坚守"和"坚守的代价"
   - 现在的内容太光鲜了，看不到挣扎
   - 如果读者看不到挣扎，就学不会怎么在自己的公司里建立价值观

3. **缺少可操作性**
   - 读完之后，读者不知道怎么在自己的公司里建立价值观
   - 不知道什么时候应该写下价值观
   - 不知道价值观冲突时怎么选
   - 不知道怎么让价值观不变成墙上的标语

### 建议：

**如果要保留这一章，必须大改。改的方向：**

1. **正面回答"小公司需要价值观吗"**
   - 答案应该是：小公司在早期不需要成文的价值观，但需要一致的决策原则
   - 什么时候应该把决策原则写成价值观？当公司超过 XX 人，或者出现了决策不一致的情况

2. **补充价值观的代价账本**
   - 量化每个价值观的成本
   - 让读者看到：坚守价值观是真金白银的损失
   - 但长期来看，这些损失换来了信任和品牌

3. **补充价值观冲突的案例**
   - 至少一个案例：黑客精神和简单冲突了，怎么选
   - 至少一个案例：客户第一和公司利益冲突了，怎么选
   - 让读者看到：价值观不是和谐的，是需要权衡的

4. **补充知识星球内部的故事**
   - 不要只讲 CEO 的决策，讲一个普通同事的故事
   - 不要只讲成功的案例，讲一个因为坚守价值观而犯错的案例
   - 让读者看到：价值观是活的，不是墙上的标语

5. **补充操作指南**
   - 怎么在小公司里建立价值观
   - 怎么让价值观不变成空话
   - 怎么在价值观冲突时做选择
   - 给读者一个 checklist

**如果不改，建议：**

把这一章拆成两章：
- 第一章：价值观是什么（现在的内容）
- 第二章：价值观怎么用（补充冲突、代价、操作指南）

或者把这一章并入"人才密度"那一章，作为其中的一节，讲"如何用价值观筛选和培养人才"。

**结论：不要砍，但必须大改。如果不改，价值有限。**
