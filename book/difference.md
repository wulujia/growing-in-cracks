# 不一样

> 如果你和别人做一样的事，就别指望有不一样的结果。
>
> —— 阿尔伯特·爱因斯坦

> 想要不可替代，就必须时刻与众不同。
>
> —— 可可·香奈儿

## 为什么要不一样

现在的软件市场拥挤不堪。独立开发者圈子里有个戏谑的说法，每个人都要做一遍“三件套”：日记、待办、记账。往往是产品比用户多，面目却都很模糊，彼此之间没什么差别。

在这种现状下，试图比别人“好一点”往往是徒劳的。用户很懒，只要旧工具还能用，他们绝不愿承担迁移的麻烦。你觉得自己优化了 10% 的体验，但在用户眼里，这根本抵消不了重新适应新工具的成本。更糟糕的是，如果和大厂做雷同的产品，注定是一场不对等的战争。他们资源无限，抄袭你的功能只需两周，甚至可以免费赠送。一旦陷入同质化，唯一的竞争维度就只剩价格。

所以，“不一样”不是为了标新立异，它是避免卷入价格战、避免成为大厂炮灰的最基本逻辑。

## 真正的不一样

第一个标准：好十倍。不是好一点，是质变。想做比 Instagram 更好的图片社交，或者比微信更好的聊天工具，没有好几倍，用户根本不会切换。

第二个标准：人无我有。找夹缝，做产业链条里的小环节。做别人不做的，或者做不好的。

但这还不够。单点的不一样太脆弱。

我在信息安全行业创业时，团队做过一个杀木马病毒的软件叫“超级巡警”。团队很有创造力，每周都有新主意，不断创新。但结局很惨。一个功能有效，行业巨头会迅速跟进，并且以他们的体量宣布这是他们首创。我们成了替巨头做预研的研发小组。还自带干粮。

功能可以抄，速度越来越快。大厂看上你的创新，两周就能上线同款，甚至免费送。真正的不一样，是个系统。

知识星球有个例子。对星主来说，能抄功能，不能抄社区。社区是时间积累出来的，是创作者一个个聚起来的，是文化慢慢长出来的。这个系统，抄不走。

微信也是。没有送达通知，骨子里是一种自信：你发了，对方就一定能收到。这背后是扎扎实实的技术内功。现在做 IM，敢夸这样的海口吗？独特不是一个灵光一现的点子，是多个环节配合的系统。

## 从不同到更好

与其更好，不如不同。与其不同，不如更好。这两句话不矛盾。

你先找到夹缝，做出差异化，避开正面竞争。这是活下来的第一步。但光有差异化不够，你还需要扎扎实实的内功。

什么是内功？

对知识星球来说，内功藏在每个环节里。速度上，访问更快一点，文件上传下载更快一点，海外访问更快一点。支付上，转化率更高一点，失败率下降一点，第一次支付更舒服一点，续费更顺畅一点。体验上，翻看历史内容更有效率一点，信息流更精准一点，消息送达更准确一点。工具上，创作工具更顺手一点，弱网络环境可用性更强一点。服务上，给星主的数据更丰富和有效一点，对企业的支持更体贴一点。好多一点可以做。累积好这些一点点，慢慢就不同了。

表面的容易抄，深层的难抄。有同行高峰期超过 1000 人，多次大额融资，做课程、做训练营、做直播，功能全面，他们在尝试做社区产品的时候，全盘照抄我们的 UI 与交互，做了个外观很接近的产品。知识星球只有 50 人，不怎么加班，只做付费社群一件事。但微信指数和百度指数上，我们也并不逊色多少。为什么？因为我们只做一件事，做到最简单。一点点优化基本功，一点点累积。这个专注和坚持，抄不走。

知识星球里还有另一个例子。经常有星主来找我们，很焦虑。他们说，我辛辛苦苦写的内容，会被人盗版。文章被复制粘贴到别的平台，音频被下载转发，甚至有人专门做爬虫，把整个星球的内容都搬走。

这确实很难受。我也承认，在当前的互联网环境下，单纯防盗版很难。技术手段总有破解方法，内容总能被复制。但我会告诉他们：内容能盗，社区盗不走。文件可以复制，文章可以搬运。但是你和用户之间的关系，用户和用户之间的互动，这些盗不走。

一个星主，如果只是单向输出内容，那他确实脆弱。内容被盗走，价值就没了。但如果他做好了用户的连接，情况就不一样了。他认识每个用户，知道他们的需求，能及时回应他们的问题。用户之间也互相认识，互相帮助，形成了网络。这些关系、连接、感情、信任，盗版者拿不走。

## 难以复制的独特

难以复制的独特，往往来自三个地方：多个环节配合形成的系统、时间积累出来的厚度、别人看不懂的认知壁垒。

知识星球有个例子。好些年里，有不少产品模仿知识星球。像素级抄袭，连产品内的图标都照搬过去。那时候，我们做了个“广场”功能。在广场里，展示知识星球里的诸多星球，各行各业非常丰富，很多星球内容更新快、付费用户多。这个功能，同行无脑抄走了。但反而暴露了他们的虚弱。没几个有价值的频道，频道里的用户也不活跃。功能照搬过去了，生态没长出来，反而抄坏了。

## 知识星球的不一样

2017 年做知识星球时，市场上已经有很多看起来相似的产品。得到、微博、知乎、36氪，都在做内容付费。我们凭什么能活下来？

答案是：找到每个产品没覆盖的缝隙，然后把这些缝隙连成一条独特的路。

得到要头部流量，签约顶级作者，做精品课程。知识星球聚长尾，服务那些有一千个铁杆粉丝的普通创作者。得到是出版社逻辑，我们是社区逻辑。

微博强调“发出我的声音，让全世界听到”。知识星球反过来，“只说给在意我和我在意的人”。微博是广场，知识星球是客厅。

知乎围绕问题组织内容，用户关注的是“这个问题下谁回答得好”。知识星球围绕人组织内容，用户关注的是“这个人说了什么”。知乎是问题崇拜，知识星球是个人崇拜。

36氪做付费专栏，用户订阅后单向接收内容。知识星球做社区，星主和用户可以互动，用户和用户之间也能交流。订阅是媒体，社区是关系。

微信是即时通讯工具，追求极速。消息要快，送达要准，所以它不会在本地存太多历史信息。知识星球反过来，做长效存储。三年前的帖子、五年前的讨论，随时能翻出来看。微信是流水，知识星球是湖泊。

更重要的是，知识星球不是要和微信竞争，而是长在微信生态里。用户在微信群里聊天，沉淀到知识星球里保存。星主可以用微信视频号做直播，只给知识星球的付费会员看。微信做它擅长的快和即时，我们做它不会做的慢和沉淀。

理解巨头和大公司的方向，能更好找到自己的位置。微信追求快、可靠、敏捷，这些方向上它会越做越好。但长效存储、社区沉淀、付费内容管理，这些不在它的主航道上。我们就在这些缝隙里生长。

每一个对比，都是一个选择。选择聚长尾而非抢头部，选择客厅而非广场，选择围绕人而非围绕问题，选择关系而非媒体。这些选择叠加起来，就形成了一个别人很难照搬的系统。

单独看每个选择，都不难理解。但要把这些选择一以贯之地执行下去，需要真的相信这条路。

## Slax Note 的不一样

在选要做的产品时，除了常规的分析，我自己往往还会琢磨两件事。一是产品能做出什么不一样——我不想只是抄，也不想仅仅做了“又一个某某”。二是我愿不愿意长期做，以及它能不能有长期的迭代进化空间。

就 Slax Note 这个小产品，最初来自一个具体的困扰。我的灵感经常来自两个场景，一是洗澡，二是散步。但这两个场景下，掏出纸笔或者手机打字记录，都有些怪异。我试过简略记几个字，但几天后回顾，常常忘了当时想的是什么。灵感就这样溜走了。所以我想做个工具：用语音快速记录，AI 帮我整理成可用的文字，方便复制到其他地方使用。

但真的开始做之前，我知道这会是个很卷的市场。传统笔记产品像 Evernote、Notion 都在叠加语音能力和 AI 能力。新生笔记产品觉得这是弯道超车的机会。还有操作系统内置的笔记在增强能力，各种会议记录产品、模型公司的全能 ChatBot 也在侵蚀这个市场。

我花了些时间调研竞品。Audio Pen 功能全面，有改写级别、写作风格、输出长度、专业词汇、第三方集成。闪念贝壳做得很漂亮，但往“问问 AI”、“待办”等方向走，想做的事情太多。Get 笔记是个大而全的产品，App、小程序、网页端都有。调研完一圈，我反而看到了机会。大家都在做多、做全，我可以做少、做简单。

这不是为了偷懒，而是基于一个判断：用户需要的不是更多功能，而是更确定的体验。打开就知道能干什么，用完就能得到想要的结果。

Slax Note 的不一样，围绕一件事展开：灵感有输入、加工、输出三个环节，我们沿着这条线，一步步往深处走。

首先是输入。界面交互尽可能简单，拿起就用。录音、输入、记录灵感，只解决这一个问题。不做日记，不做待办，不做会议纪要。打开 App，说几句话，灵感就留住了。

产品上线后，我自己成了重度用户。写公众号的流程变了。以前坐下来打字、改，一篇文章要花 1 个小时。现在打开 Slax Note，看看记录过哪些灵感，挑一个，重听录音，脑子里梳理一下，对着手机直接说。10 分钟搞定。灵感不丢了，甚至感觉说话能力也在变好——录音多了，边想边说，语言组织能力在提升。

然后是加工。用户记录了灵感，接下来要把灵感变成有用的东西。加工的方式可以由浅入深：最简单的是 Prompt，用户选一个 Prompt，AI 把语音转写的内容润色成不同风格的文字。再往上是 Skill，把多个步骤组合成一个技能，比如“把一段灵感变成小红书帖子+朋友圈文案+知识星球主题帖”。再往后，说不定是多模态的各种生产。

一位同事跟我聊，他说自己没有语音笔记的习惯，对 Slax Note 没什么需求。但他说，如果能变成他的“智囊团”，他可能会用。想到一个主意，语音记录下来，然后用不同的方式激发灵感，对一个观点做发散思考。这个反馈让我确定了方向：加工能力是 Slax Note 的第二个战场。

最后是沉淀。输入和加工解决了“从灵感到内容”的问题，但更大的价值在于积累。我们会做一个社区，让用户的 Prompt 和 Skill 可以相互交流。大家可以看到彼此是怎么使用这个产品的，学习和使用别人的技巧。绝大多数人不知道怎么写好一个 Prompt，但如果有个地方可以看到别人的玩法，可以一键套用，可以互相启发，这个社区就会慢慢长出价值。

这个价值需要时间，需要用户一个个玩法的积累。功能可以抄，社区和积累难抄。

所以相比市面上的会议录音产品或者读书笔记产品，Slax Note 走的是另一条路。它会一直做得很小、很简单、很快，但沿着输入、加工、沉淀这条线，长期做下去。

更难抄的是个性和认知。做简单而非大而全，回到问题本质而非追求酷的技术，用系统而非功能点来思考产品。这些判断和取舍，藏在产品的每个决策里。独特不是灵光一现，而是个系统工程。
